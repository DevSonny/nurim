# Nurim — MVP Spec

## 1. 문제

자기개발을 하고 싶은 사람들이 기존 도구(노션, 습관 트래커)를 쓰면 혼자만 보는 기록이라 동기부여가 약하고 하루를 돌아보는 느낌이 없다. 인스타는 자랑은 되지만 성장 데이터가 없다. 오타니 만다라트에서 영감을 받았지만 만다라트는 고정 구조라 유연하지 않다. 매일매일 커가는 시각적 성장과 커뮤니티 동기부여를 동시에 주는 서비스가 없다.

## 2. 사용자 (페르소나)

자기개발에 관심 있고 기록하고 싶은 사람. 갓생 콘텐츠를 즐겨보고, 성장을 인스타·페이스북에 자랑하고 싶은 20대. UI가 예쁘면 공유 욕구가 생기는 사람.

## 3. 도메인 용어

| 용어 | 설명 |
|------|------|
| **Core** | 사용자 중심 노드. 달성된 목표를 흡수하며 연간 단위로 성장함 |
| **Orbit** | Core에 붙는 목표 노드 (예: 공부, 운동) |
| **Sub-Orbit** | Orbit에 붙는 세부 목표 노드 (예: 영어공부 → 듣기/쓰기/말하기) |
| **Pulse** | 하루 1회 Sub-Orbit에 찍는 숫자 기록. 소급 입력 허용, 양심 시스템 |
| **Proof** | 분자 그래프를 커뮤니티 피드에 공유하는 게시물 |

## 4. 핵심 기능 (MVP — 30일)

### Week 1~2: 기반
- [ ] Google OAuth 로그인
- [ ] Core 생성 (사용자당 1개, 연간 단위)
- [ ] Orbit 자유 추가/삭제 (최대 8개)
- [ ] Sub-Orbit 자유 추가/삭제 (Orbit당 최대 8개)
- [ ] 누적형 / 반복형 목표 설정
- [ ] 날짜별 Pulse 입력 (소급 입력 허용)

### Week 3: 시각화 + 통계
- [ ] D3 force simulation 분자 그래프 (Core → Orbit → Sub-Orbit)
- [ ] 노드 크기 = 누적 기록량, 색상 채움 = 최근 7일 활성도
- [ ] Framer Motion 성장 애니메이션 (Pulse 입력 시 노드 뿅 커짐)
- [ ] 목표 달성 시 노드가 Core에 흡수되는 애니메이션
- [ ] 통계: 스트릭, 진행률, 누적 그래프

### Week 4: 커뮤니티
- [ ] Proof 공유 (게시물 단위 공개)
- [ ] 이모지 반응 + 댓글
- [ ] 팔로우/팔로워
- [ ] 인스타/페이스북 공유 버튼 (og:image 자동 생성)

## 5. 연간 펫 시스템

- 분자는 **연간 단위**로 성장한다
- 연말에 리셋 → 새해에 새 분자(펫)를 키우기 시작
- 달성된 Sub-Orbit/Orbit은 Core에 흡수 → Core가 점점 커짐
- 지난 연도 분자는 아카이브로 보존 (내 성장 히스토리)

## 6. 제외 범위 (이번엔 안 함)

- 모바일 앱 (웹 + PWA만)
- Sub-Orbit 하위 4단계 이상
- 카카오/애플 소셜 로그인
- 푸시 알림 (PWA 푸시는 v2)
- Orbit 간 연결/의존 관계

## 7. 성공 기준 (데모 시나리오)

1. 구글 로그인
2. Core 생성 → Orbit "공부" 추가 → Sub-Orbit "영어듣기/쓰기/말하기" 추가
3. Pulse 입력 → 노드가 커지는 애니메이션 확인
4. 자격증 달성 → Core 흡수 애니메이션 확인
5. Proof 공유 → 다른 계정에서 이모지 반응
6. 인스타 공유 버튼으로 분자 이미지 공유

## 8. 기술 스택

| 레이어 | 선택 | 이유 |
|--------|------|------|
| Frontend | Next.js + D3.js + Framer Motion | 기존 프로젝트 + 동적 애니메이션 |
| Backend | Cloudflare Workers + Hyperdrive | 무료 사용량 풍부, 새 스택 경험 |
| DB | Oracle Cloud PostgreSQL | Always Free, 넉넉한 사양 |
| Auth | Google OAuth | 가입 허들 최소화 |
| Deploy | Cloudflare Pages | Workers와 네이티브 통합 |

## 9. 일정 (30일)

| 기간 | 목표 |
|------|------|
| Day 1~3 | 스택 세팅 (Cloudflare + Oracle + Google Auth) |
| Day 4~10 | Core/Orbit/Sub-Orbit CRUD + Pulse 입력 |
| Day 11~20 | D3 분자 시각화 + Framer Motion 애니메이션 |
| Day 21~25 | 통계 + 스트릭 + 달성 흡수 애니메이션 |
| Day 26~30 | Proof 피드 + 이모지/댓글 + SNS 공유 |
