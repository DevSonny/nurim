# Nurim PRD

## Problem Statement

현재 학습/성장 기록 도구들은 각각 약점을 가지고 있습니다.

- **Notion/스프레드시트**: 혼자만 보는 데이터, 동기부여 약함
- **습관 트래커**: 성장 가시성 낮음, 커뮤니티 없음
- **인스타그램**: 자랑은 되지만 객관적 성장 데이터 부족
- **만다라트**: 고정 구조, 유연성 부족

사용자는 **"눈 앞에서 보이는 성장" + "공유하고 싶은 커뮤니티"**를 동시에 원하지만, 이를 만족하는 도구가 없습니다.

## Solution

Nurim은 학습 목표를 **시각화된 분자(노드)**로 표현하고, 매일의 기록(Pulse)이 쌓이면서 노드가 자라는 것을 실시간으로 보여주는 성장 추적 앱입니다.

- **시각 피드백**: 기록하는 즉시 분자가 자라는 애니메이션으로 즉각적 동기부여
- **계층적 목표**: Core(연단위 큰 목표) → Orbit(중간 목표) → Sub-Orbit(작은 단계)로 자유롭게 설계
- **사회적 검증**: 달성 시 분자 흡수 애니메이션 후 커뮤니티에 공유(Proof) 가능, 이모지 반응으로 자연스러운 검증
- **자동 리셋**: 연말 자동 리셋으로 새로운 사이클 시작

**핵심 데모 흐름**:  
Google 로그인 → Core 생성 → Orbit/Sub-Orbit 추가 → Pulse 입력 → 노드 성장 애니메이션 → 달성 목표 흡수 → Proof 공유 → 이모지 반응

## User Stories

1. As a 20대 직장인, I want to 연간 목표를 입력하고 그에 따른 세부 목표를 자유롭게 추가할 수 있어야, so that 내 목표를 유연하게 관리할 수 있습니다.

2. As a 학습자, I want to 공부 시간/책 읽은 페이지/운동 횟수 등 다양한 단위로 진전도를 기록해야, so that 서로 다른 목표를 공통의 단위 없이 추적할 수 있습니다.

3. As a 사용자, I want to Pulse를 입력하면 화면에서 분자가 자라는 애니메이션을 즉시 봐야, so that 기록의 즉시적 보상을 느낄 수 있습니다.

4. As a 사용자, I want to 달성 목표가 자동으로 인식되거나 수동으로 버튼을 눌러 달성 표시해야, so that 성취감을 명확히 느낄 수 있습니다.

5. As a 달성한 사용자, I want to 달성 시 분자가 흡수되는 애니메이션을 봐야, so that 이 목표가 완료되고 다음 단계로 넘어감을 시각적으로 이해할 수 있습니다.

6. As a 사용자, I want to 달성한 목표나 분자 성장 과정을 커뮤니티에 공유(Proof)할 수 있어야, so that 다른 사용자와 함께 성장하는 경험을 할 수 있습니다.

7. As a 커뮤니티 참여자, I want to 다른 사용자의 성장 기록에 이모지로 반응할 수 있어야, so that 같은 길을 가는 사람들과 자연스러운 응원을 나눌 수 있습니다.

8. As a 사용자, I want to 내 프로필에서 올해의 성장 현황을 한눈에 보는 대시보드가 있어야, so that 연간 성장 궤적을 파악할 수 있습니다.

9. As a 사용자, I want to 구글 로그인으로 간편하게 가입 및 로그인해야, so that 복잡한 가입 절차 없이 바로 시작할 수 있습니다.

10. As a 신규 사용자, I want to 직관적인 온보딩을 통해 Core/Orbit/Sub-Orbit이 무엇인지 이해해야, so that 처음부터 도구를 올바르게 사용할 수 있습니다.

11. As a 사용자, I want to 카테고리 프리셋(공부, 운동, 독서, 언어, 자격증·시험, 창작, 루틴·습관, 커리어, 기타)에서 선택하거나 직접 정의해야, so that 목표 생성이 빠르고 일관됩니다.

12. As a 사용자, I want to 각 Sub-Orbit마다 단위를 직접 입력(시간, 페이지, 회, 분, km, 단어 수, 취득 등)할 수 있어야, so that 다양한 활동을 통일된 형식으로 추적할 수 있습니다.

13. As a 사용자, I want to 내 분자 성장 데이터가 실시간 2D/3D 시각화로 표현되어야, so that 추상적인 진전도를 구체적으로 느낄 수 있습니다.

14. As a 사용자, I want to 연말이 되면 Core가 자동으로 리셋되고 새로운 사이클이 시작되어야, so that 매년 새로운 목표를 설정할 수 있습니다.

15. As a 커뮤니티 멤버, I want to 피드에서 다른 사용자들의 성장 기록(Proof)을 볼 수 있어야, so that 영감을 받고 함께 성장하는 느낌을 받을 수 있습니다.

16. As a 사용자, I want to 공유한 Proof에 달린 이모지 반응 수를 볼 수 있어야, so that 커뮤니티의 응원 정도를 알 수 있습니다.

17. As a 사용자, I want to 내가 기록한 Pulse를 언제든 수정하거나 삭제할 수 있어야, so that 오류 기록을 바로잡을 수 있습니다.

18. As a 사용자, I want to 목표 달성에 필요한 Pulse 누적값을 설정할 수 있어야, so that 명확한 달성 기준을 정할 수 있습니다.

19. As a 사용자, I want to 달성하지 못한 목표도 포기하지 않고 계속 진행 중으로 표시할 수 있어야, so that 실패감 없이 지속적으로 추적할 수 있습니다.

20. As a 신규 사용자, I want to 데모 데이터(샘플 Core/Orbit/Sub-Orbit)로 앱이 어떻게 작동하는지 먼저 체험해야, so that 가입 전에 가치를 이해할 수 있습니다.

## Implementation Decisions

### 1. Core/Orbit/Sub-Orbit 데이터 모델

- **Core**: 연단위 최상위 목표 (예: "2026년 성장")
- **Orbit**: Core 하의 중간 목표 (예: "영어 마스터")
- **Sub-Orbit**: Orbit 하의 실행 단위 (예: "영어 책 읽기" → 목표: 10권)
- Sub-Orbit 하의 추가 계층은 v2에서 검토

### 2. Pulse 입력 및 단위

- 사용자가 카테고리를 선택하거나 직접 정의 가능
- 프리셋 카테고리: 공부 / 운동 / 독서 / 언어 / 자격증·시험 / 창작 / 루틴·습관 / 커리어 / 기타
- 단위는 자유 입력: 시간, 페이지, 회, 분, km, 단어 수, 취득 등
- Pulse 기록은 **자기 신고** 방식 (검증 메커니즘 없음)
- Proof(커뮤니티 공유)가 자연스러운 사회적 검증 역할

### 3. 달성 트리거 2가지

1. **수동 달성**: 사용자가 "달성" 버튼을 눌러 명시적 표시
2. **자동 달성**: Pulse 누적값이 Sub-Orbit의 목표치 도달 시 자동 인식

달성 시 → Core 흡수 애니메이션 발생

### 4. 시각화 기술 스택

- **2D 시각화**: D3.js force simulation (기본)
- **3D 시각화**: Three.js + React Three Fiber (선택 사항, 기존 구현 활용)
- **애니메이션**: 
  - Pulse 입력 시 노드 성장 애니메이션
  - 달성 시 Core 흡수 애니메이션
  - 분자 이동/상호작용 물리 시뮬레이션

**핵심 검증 항목**: "분자가 자라고 흡수되는 시각화가 만족스러운 품질로 구현되는가" (주 3의 최우선 항목)

### 5. 인증 및 데이터 저장

- **인증**: Supabase Auth (Google OAuth 통합)
- **데이터베이스**: Supabase PostgreSQL
- **세션 관리**: Supabase 클라이언트 라이브러리

### 6. 백엔드 API 구조

- **프레임워크**: Next.js API Routes
- **필수 엔드포인트**:
  - `POST /api/core` - Core 생성
  - `GET /api/core/:id` - Core 상세 조회
  - `POST /api/orbit` - Orbit 생성
  - `POST /api/sub-orbit` - Sub-Orbit 생성
  - `POST /api/pulse` - Pulse 기록
  - `PATCH /api/sub-orbit/:id/achieve` - 목표 달성 표시
  - `POST /api/proof` - Proof 공유
  - `POST /api/proof/:id/reaction` - 이모지 반응 추가

### 7. 커뮤니티 피드 기능

- **Proof 공유**: 사용자가 달성한 목표나 성장 과정을 커뮤니티에 텍스트 + 스크린샷으로 공유
- **이모지 반응**: 최소 5개 이모지 (👍, 🔥, 🎉, 💪, 🌟) 로 반응 가능
- **댓글**: v2에서 검토 (현재 미포함)
- **팔로우/팔로워**: v2에서 검토 (현재 미포함)

### 8. 배포 및 인프라

- **프론트엔드**: Vercel (Next.js 기본 호스팅)
- **데이터베이스**: Supabase (PostgreSQL)
- **OG 이미지 생성**: `@vercel/og` 라이브러리 활용 (사소한 기능, 우선순위 낮음)

### 9. 스택 선택 근거

| 레이어 | 선택 | 이유 |
|--------|------|------|
| Backend | Next.js API Routes | 처음 쓰는 스택(Cloudflare Workers) 제거, 기존 경험 활용 |
| DB | Supabase PostgreSQL | 무료티어 안정성, Auth 통합, 빠른 세팅 |
| Auth | Supabase Auth + Google OAuth | DB와 단일 통합, 세팅 간단 |
| Deploy | Vercel | Next.js 네이티브, 무료티어 충분 |
| Visualization | D3.js + Three.js | D3로 기본 구현, Three.js로 고급 표현 |

### 10. 온보딩 및 UX

- **Google 로그인 후 첫 화면**: Core 생성 유도
- **도움말**: Core/Orbit/Sub-Orbit 개념 설명 모달
- **데모 데이터**: 옵션으로 샘플 생성 후 실제 데이터 입력으로 전환
- **빈 상태(Empty State)**: 각 페이지에서 첫 항목 생성 유도

## Testing Decisions

### 테스트의 원칙

- 외부 동작만 테스트 (구현 상세는 테스트하지 않음)
- 시각적 피드백이 핵심이므로 UI 통합 테스트에 무게
- 데이터 일관성은 백엔드 엔드-투-엔드 테스트로 검증

### 테스트 대상 모듈

1. **인증 플로우**
   - Google OAuth 로그인 성공
   - 이미 로그인한 사용자 재접속
   - 로그아웃 및 세션 만료

2. **Core/Orbit/Sub-Orbit CRUD**
   - Core 생성 → Supabase에 저장 확인
   - Orbit/Sub-Orbit 추가 시 계층 구조 유지 확인
   - 수정/삭제 시 하위 항목 상태 확인

3. **Pulse 입력 및 시각화**
   - Pulse 입력 → DB 저장 → 화면에 노드 성장 애니메이션 즉시 표시
   - 누적값 확인 (정확한 수치)
   - 단위 변환 (시간 → 분 등) 올바름

4. **달성 트리거**
   - 수동 달성 버튼 클릭 → Core 흡수 애니메이션 표시
   - 자동 달성 (누적값 도달) → 즉시 인식 및 애니메이션

5. **Proof 공유 및 반응**
   - Proof 작성 → 커뮤니티 피드에 표시
   - 이모지 반응 추가 → 실시간 카운팅 업데이트
   - 피드 로드 성능 (무한 스크롤 시 지연 없음)

6. **데이터 정합성 (Supabase)**
   - 트랜잭션: Pulse 기록 시 누적값과 달성 상태 동시 업데이트
   - 동시성: 여러 디바이스에서 동시 입력 시 충돌 없음
   - 삭제: Pulse 삭제 시 누적값 재계산 및 달성 상태 되돌림

### 참고: 테스트 마이그레이션

현재 프로젝트에 기존 테스트 패턴이 없으므로, 구현 과정에서 다음을 참고:
- Jest + React Testing Library (가장 일반적)
- Playwright (E2E, 특히 애니메이션 시각 검증)
- Supabase 테스트 데이터베이스 (각 테스트 후 초기화)

## Out of Scope (v1)

### 지연된 기능 (v2에서 검토)

- **Sub-Orbit 하위 4단계 이상**: 현재 3단계(Core/Orbit/Sub-Orbit)만 지원
- **모바일 네이티브 앱**: 웹 + PWA만 제공 (반응형 모바일 웹)
- **SNS 공유**: 카카오톡, 트위터 등 외부 채널 공유 (OG 이미지 자동 생성만 구현)
- **댓글 기능**: Proof에 댓글 추가 불가 (이모지 반응만 지원)
- **팔로우/팔로워**: 사용자 간 관계 기능 없음
- **푸시 알림**: 웹 푸시 알림 미지원
- **카카오/애플 로그인**: Google OAuth만 지원
- **오프라인 모드**: 온라인 전용

### 아키텍처 제약 (v1 규칙)

- 양심 시스템: Pulse 입력에 검증 메커니즘 없음 (자기 신고 방식)
- 고급 통계: 연간 성장률 차트, 자세한 분석 등은 미포함
- 목표 공동 편집: 개인 목표만 지원 (협업 목표는 v2)

### 도메인 용어 정의

| 용어 | 정의 | 예시 |
|------|------|------|
| **Core** | 연단위 최상위 목표 | "2026년 성장" |
| **Orbit** | Core 하의 중간 목표 | "영어 마스터" |
| **Sub-Orbit** | Orbit 하의 실행 단위 | "영어 책 읽기" |
| **Pulse** | Sub-Orbit 진전도 기록 | "오늘 30분 공부", "책 2페이지" |
| **Proof** | 달성 또는 성장 과정을 커뮤니티에 공유 | "영어 책 5권 달성!" |
| **노드/분자** | 시각화된 목표 표현 | D3/Three.js로 화면에 표시 |
